Project Idea

Overview

My project is a website aggregator that will display the schedule, news, and race data related to Formula 1 in a single webpage with interactive components. The name of the project that I have chosen is F1Dash. It is a play on the word “dash” which means to be in a hurry (something you want to be as a driver in Formula1) as well as a shortened version of the word dashboard. The interactive components will allow a user to customise the data visualizations which can be used for analytics and comparison.

Motivation

Globally, the average audience per Grand Prix in 2021 was 70.3m so there is a large market for an application that aggregates Formula 1 news coupled with an interactive statistics component. I am an avid Formula 1 fan and love to keep up to date on the latest news, race statistics, and data but I must go to multiple sources and webpages in my bookmarks for the variety of Formula 1 related topics that I am interested in. The goal of this project is to display it all in one place using modern open-source UI components and visualization libraries to make the application look beautiful.

Description

The application will have 2 main components. The first section will comprise of an aggregation of the latest Formula 1 news articles and social media posts from various online sources that would be displayed in a chronological timeline. This data will be obtained by integrating with available API’s. If no API’s are available, there will be services that scrape a list of websites for the relevant data to use in the app. The application will not be ripping off the content from the sources to display as a whole. Instead it will provide a short summary of the article with a link to the original source. The second section will be an interactive page where a user can view previous race and telemetry data using a range of front-end components to filter the data to their liking. This would allow a user to perform analysis and comparisons of the data for their own entertainment purposes.
